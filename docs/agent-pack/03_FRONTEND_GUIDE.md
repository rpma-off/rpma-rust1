│       ├── hooks/               # Custom React hooks

│       ├── services/            # Client-side business logic
│       └── stores/              # Zustand stores (calendar, notifications)
├── lib/
│   ├── ipc/                     # Core IPC engine
│   │   ├── utils.ts             # safeInvoke (core engine, 450+ lines)
│   │   ├── client.ts            # ipcClient singleton + useIpcClient hook
│   │   ├── cache.ts             # TTL request cache + invalidation
│   │   ├── commands.ts          # IPC_COMMANDS registry (100+ names)

│   │   └── domains/             # Domain-namespaced operation bundles
│   ├── query-keys.ts            # Centralized TanStack Query key factories
│   └── rbac.ts                  # Frontend permission matrix (26 permissions)

## Provider Hierarchy

```
layout.tsx (Server)
└── RootClientLayout.tsx (Client)
    └── Providers
        ├── QueryClientProvider      (TanStack Query — staleTime 5min, gcTime 10min)
        │   ├── MutationSignalListener
        │   └── AuthProvider
        │       ├── NotificationInitializer
        │       ├── {children} (page content)
        │       ├── Toaster (Sonner, top-right, richColors)
        │       └── NotificationPanel
        └── ThemeProvider (light/dark auto)
```

`AppLayout` (inside RootClientLayout) handles:
- Auth state checks via `useAuth()`, `useAuthRedirect()`, `useAdminBootstrapCheck()`
- Tauri event listener (`useTauriEvent`) for backend→UI invalidation
- Menu events, system logging init
- Hydration guard (skeleton until auth ready)

## State Management

| Type | Tool | Usage |
|------|------|-------|
| Server / async state | TanStack Query | All backend data: `useQuery` + `useMutation` |
| Global UI state (calendar) | Zustand + `persist` | `domains/calendar/stores/calendarStore.ts` — persisted to localStorage, synced to backend on debounce |
| Global UI state (notifications) | Zustand | `domains/notifications/stores/notificationStore.ts` — panel open/closed, WS status |
| Component state | React `useState` | Form fields, local toggles |

**TanStack Query config** (set in `providers.tsx`):
```typescript
staleTime: 5 * 60 * 1000     // 5 min
gcTime: 10 * 60 * 1000       // 10 min
refetchOnWindowFocus: false
refetchOnReconnect: true
```

## IPC Pattern (How Frontend Calls Backend)

**Never call `invoke()` directly.** Always use the typed wrappers.

### Core engine: `safeInvoke` (`lib/ipc/utils.ts`)
Automatically handles:
- Session token injection (from `AuthProvider`)
- Correlation ID injection (for end-to-end tracing)
- Timeout enforcement (15s default)
- Error normalization (AppError codes → French user messages)
- Retry logic, metrics recording, logging

### ipcClient singleton (`lib/ipc/client.ts`)
```typescript
import { ipcClient } from '@/lib/ipc';

// Read
const task = await ipcClient.tasks.get(id);

// Mutation
await ipcClient.tasks.create(payload);
```

The client bundles all domain wrappers:
```typescript
export const ipcClient = {
  auth, tasks, clients, interventions, photos, ppfWorkflow,
  material, inventory, calendar, notifications, quotes, reports,
  users, settings, admin, bootstrap, rules, integrations, trash, system
} as const;
```

### Domain IPC wrapper pattern
```typescript
// domains/tasks/ipc/task.ipc.ts
export const taskIpc = {
  create: (data) => safeInvoke('task_create', data),
  get: (id) => cachedInvoke(`task:${id}`, 'task_get', { id }, validateTask),
  list: (filters) => safeInvoke('task_list', filters),
  updateStatus: (id, status) => safeInvoke('update_task_status', { id, status }),
};
```

### Caching strategy
- `cachedInvoke(key, command, args, validator, ttl)` — in-memory TTL cache (default 60s)
- `invalidatePattern('task:')` — prefix-based cache bust on writes
- `signalMutation('tasks')` — triggers TanStack Query refetch via `MutationSignalListener`

## Domain Boundary Rule
Each domain exposes its public surface through `api/index.ts`:
```typescript
import { useAuth } from '@/domains/auth/api';         // ✅ correct
import { useAuth } from '@/domains/auth/api/AuthProvider'; // ❌ leaks internals
```
Internal domain files use **relative imports** to avoid circular dependencies.

## Query Keys (`lib/query-keys.ts`)
Centralized factories for cache invalidation:
```typescript
taskKeys.lists()               // all task list queries
taskKeys.byId(id)              // single task
interventionKeys.byTask(taskId)
interventionKeys.ppfWorkflow(taskId)
adminKeys.appSettings()
```

## Adding a New UI Feature
1. Identify the domain → place code in `frontend/src/domains/<domain>/`
2. Add IPC wrapper in `domains/<domain>/ipc/<domain>.ipc.ts`
3. Add query keys to `lib/query-keys.ts`
4. Build `useQuery`/`useMutation` hook in `hooks/`
5. Build UI using shadcn/ui components + Tailwind
6. Use Zod for form validation before IPC calls
7. Export from `api/index.ts`

## Common Pitfalls
- **Type drift**: After any Rust struct change, run `npm run types:sync` or the frontend build fails.
- **IPC command name**: The string in `safeInvoke('command_name')` must exactly match the `#[tauri::command]` name in Rust.
- **No raw invoke**: Do not import or call `@tauri-apps/api/tauri` `invoke` directly in domain code.
- **Query key consistency**: Always use `query-keys.ts` factories — hardcoded strings cause stale cache bugs.
