## Context

The application uses Tauri for IPC between a Rust backend and TypeScript frontend. Type mismatches between backend and frontend cause:

- Runtime errors from unexpected data shapes
- Manual type duplication leading to drift
- Brittle refactoring when models change
- Increased onboarding time for type maintenance

Maintaining two parallel type definitions (Rust and TypeScript) is error-prone and violates DRY principles.

## Decision

**Use ts-rs to auto-generate TypeScript types from Rust models.**

### Implementation

Rust models derive `TS` trait from ts-rs (`src-tauri/src/bin/export-types.rs`):

```rust
use ts_rs::TS;

#[derive(Serialize, Deserialize, TS)]
#[ts(export)]
pub struct Task {
    pub id: String,
    pub title: String,
    pub status: TaskStatus,
    // ...
}
```

The export binary (`src-tauri/src/bin/export-types.rs`) generates types to `frontend/src/lib/backend/`:

```typescript
// AUTO-GENERATED. DO NOT EDIT.
// Generated by src-tauri/bin/export-types + scripts/write-types.js

export * from './auth';
export * from './calendar';
export * from './clients';
// ... 16 domain modules
```

### Type Synchronization Commands

Defined in `package.json`:

```bash
npm run types:sync       # Generate types from Rust
npm run types:validate   # Verify generated types are current
npm run types:drift-check # Detect manual modifications
npm run types:watch      # Regenerate on file changes
```

### Special Handling

- **serde_json::Value**: Handled via `#[ts(type = "JsonValue")]` attributes
- **Optional fields**: Mapped to TypeScript optional properties
- **Enums**: Generated as TypeScript union types or enums based on configuration

## Consequences

### Positive

- **Single Source of Truth**: Rust models define the contract
- **Compile-Time Safety**: TypeScript catches type mismatches at build time
- **Refactoring Confidence**: Changing Rust models updates TypeScript automatically
- **Documentation**: Generated types serve as API documentation
- **Reduced Boilerplate**: No manual TypeScript interface maintenance

### Negative

- **Build Complexity**: Additional build step required for type generation
- **Limited Features**: ts-rs may not support all advanced TypeScript patterns
- **Indirect Debugging**: Type errors may require checking Rust source
- **Attribute Overhead**: Complex types require ts-rs attributes
